# Memoir Clipper

Memoir Clipper is a browser extension designed to capture text, images, and links from any web page while preserving formatting and context. The project leverages TypeScript, Vite, and Tailwind CSS, and is structured for modularity and maintainability.

---

**Branching Policy**

- The `develop` branch contains the latest changes and active development work.
- The `main` branch is the latest stable branch and should be used for production releases.

## Installation & Development

### Prerequisites

- Node.js (v18 or higher recommended)
- npm (v9 or higher recommended)

### 1. Install Dependencies

```bash
npm install
```

### 2. Available Scripts

Below are all available npm scripts for development, building, and maintenance:

| Script              | Description                                                                                   |
|---------------------|-----------------------------------------------------------------------------------------------|
| `npm run dev`       | Start Vite in development mode with hot reload for extension development.                     |
| `npm run build`     | Type-checks with TypeScript and builds the extension in development mode.                     |
| `npm run watch`     | Builds the extension in development mode and watches for changes.                             |
| `npm run build:dev` | Type-checks and builds the extension in development mode (alias for `build`).                 |
| `npm run build:prod`| Type-checks and builds the extension for production (output in `dist`).                       |
| `npm run lint`      | Runs ESLint with Prettier integration to check code quality and formatting.                   |
| `npm run preview`   | Previews the built extension (where supported by Vite).                                       |

### 3. Start Development Server (with Hot Reload)

Builds the extension in development mode.

```bash
npm run dev
```

### 4. Load the Extension in Chrome

1. Open Chrome and go to `chrome://extensions/`.
2. Enable **Developer mode** (toggle in the top right).
3. Click **Load unpacked**.
4. Select the `dist` folder generated by the build scripts in your project root.

### 5. Production Build

Generates a production-ready build in the `dist` directory.

```bash
npm run build:prod
```

### 6. Lint & Format

Checks code quality and formatting.

```bash
npm run lint
```

## Extension Entry Points

- **Background**: `background/index.ts` (registered in `manifest.json`)
- **Content Script**: `content/index.ts` (injected on all pages)
- **Popup**: `popup/index.html` (browser action)
- **Options**: `options/index.html` (options page)

## Environment Variables

Environment-specific settings are managed via `.env.development` and `.env.production` files.

- `VITE_APP_ENV` — Specifies the environment (`development` or `production`)
- `VITE_LOG_ENABLED` — Enables or disables logging
- `VITE_MINIMUM_LOG_LEVEL` — Sets the minimum log level (`DEBUG`, `INFO`, `WARN`, `ERROR`)

## File & Folder Responsibilities

### `background/`
- `index.ts`: Registers context menus and manages extension events.
- `contextMenus/`: Defines and handles context menu creation and interactions.

### `content/`
- `index.ts`: Entry point for content scripts; injects styles and initializes orchestrators.
- `managers/`: Handles UI orchestration, toolbars, shortcuts, and menu management.
- `helpers/`: Provides utilities for selection analysis, HTML cleanup, and context detection.

### `models/`
- `textModel.ts`, `imageModel.ts`, `linkModel.ts`, `pageModel.ts`: Define data models for captured content.
- `extensionMessage.ts`, `extensionReponse.ts`: Specify messaging structures for background/content communication.
- `eventManager.ts`: Utility for managing event listeners.

### `popup/` & `options/`
- `index.html`, `index.ts`: Implement the popup and options page UIs.

### `styles/`
- `tailwind.css`: Tailwind base stylesheet.
- `tailwindInjector.ts`: Dynamically injects Tailwind into content scripts.
- `buttonStyles.ts`, `dropdownStyles.ts`, `inlineToolbarStyles.ts`, `toggleStyles.ts`: Isolated CSS for UI components.

### `utils/`
- `helpers/`: DOM utilities, favicon detection, logging, environment detection, and selection validation.
- `values/`: Contains constants, enums, IDs, strings, HTML tags/attributes, and types.
- `ui/`: Factories and base logic for dropdowns, toggles, and buttons.

### Root Files

- `manifest.json`: Chrome extension manifest (MV3).
- `vite.config.ts`: Vite build and alias configuration.
- `tsconfig.json`: TypeScript configuration.
- `tailwind.config.js`: Tailwind CSS configuration.
- `postcss.config.js`: PostCSS configuration.
- `package.json`: Project scripts and dependencies.

## Key Packages & Technologies

- **TypeScript**: Ensures type safety and modern JavaScript features.
- **Vite**: Provides fast, efficient builds for web extensions.
- **Tailwind CSS**: Utility-first CSS framework for rapid UI development.
- **@crxjs/vite-plugin**: Integrates Chrome extension tooling with Vite.
- **ESLint & Prettier**: Enforces code quality and consistent formatting.
- **Chrome Extension APIs**: Enables context menus, messaging, storage, and more.
